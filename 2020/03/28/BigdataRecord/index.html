原因是安装SSH的时候没有设置免密登陆

首先使用ssh localhost 登录，如果此时需要输入用户密码才能登入，确实没有设置无密码登录，那么接下来就需要再生成密钥并将其加入到授权中。

2.关于tmp文件夹自动删除问题

之前有把下载的文件放入临时文件夹的情况，第二天再登录时怎么也找不到要解压的文件，显示文件丢失。

Ubuntu系统下/tmp目录文件重启后会自动清理，它是用来存放一些临时文件的，重要的文件最好不要放在其中！

进入/etc/default/rcS 文件中查看：

发现有一项TMPTIME=0 ，0表示重启后删除文件，设置为-1时不会自动删除文件，为正整数时表示文件的保留时间，可以根据自己的需求进行相应的设置。

5.设置Linux环境变量的方法

设置 Linux 环境变量可以通过 export 实现，也可以通过修改几个文件来实现。

全局环境变量，对所有用户都会生效

  • etc/profile :系统的每个用户设置环境信息.当用户登录时，该文件被执行一次，并从 /etc/profile.d 目录的配置文件中搜集shell 的设置。一般用于设置所有用户使用的全局变量。
  • /etc/bashrc :当 bash shell 被打开时，该文件被读取。也就是说，每次新打开一个终端 shell，该文件就会被读取。

只对单个用户生效的对应方法

  • ~/.bash_profile 或 ~/.profile :只对单个用户生效，当用户登录时该文件仅执行一次。用户可使用该文件添加自己使用的 shell 变量信息。另外在不同的LINUX操作系统下，这个文件可能是不同的，如果存在几种的话，那么执行的顺序便是：~/.bash_profile、 ~/.bash_login、 ~/.profile。

    当前我是用的Ubuntu 系统一般是 ~/.profile 文件。

  • ~/.bashrc:只对单个用户生效，当登录以及每次打开新的 shell 时，该文件被读取。

修改 /etc/environment 这个文件也能实现环境变量的设置

  • /etc/environment 设置的是整个系统的环境，而/etc/profile是设置所有用户的环境。
  • /etc/environment 中直接通过 VAR="..." 的方式设置
  • 使用 source /etc/environment 可以使变量设置在当前窗口立即生效，需注销/重启之后，才能对每个新终端窗口都生效。

1.Ubuntu下Mysql5.7导入中文乱码问题

原因是character_set_server默认设置为latin1，需要将其修改为utf8格式。

  • 首先进入数据库，查看当前编码

可以看到当前character_set_server的值为latin1，需要将其修改为utf8编码格式。

  • 方法一：可以在当前终端执行set character_set_server=utf8命令，但是数据库重启后就无效了，不建议使用。

  • 方法二：修改my.cnf配置文件

  • 保存后重启Mysql
  • 重新登录查看编码，character-set-server已变为utf8格式。此时再重新进行表的导入，中文字符显示正常。

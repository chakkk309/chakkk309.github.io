Hexo Installation

最新重装了一下系统，一夜回到解放前。还好存了hexo备份，配置文件还在。但是以防万一再遇到类似窘境，还是记录一下找回过程吧~

安装node.js

安装git

安装hexo

  • 使用npm安装hexo
npm install hexo-cli -g
  • 选好一个blog文件夹，进行初始化
hexo init blog

  • 使用备份好的文件进行覆盖/替换

  • 安装部署插件

npm install hexo-deployer-git –save

配置git信息

  • blog文件夹中右键 ->Git Bash Here ，设置信息（用户名和密码）

    这里也可以将其参数设置为global，表示这台电脑上所有的Git仓库都会使用这个配置，也可以对某个仓库指定不同的用户名和Email地址。

    git config  (--global) user.name "你的GitHub用户名"
git config  (--global) user.email "你的GitHub注册邮箱"

配置SSH

  • 生成ssh秘钥，直接Enter*3默认不需要密码。

    ssh-keygen -t rsa -C "你的GitHub注册邮箱"

  • 找到生成的.ssh的文件夹中的id_rsa.pub密钥，将内容复制，进入https://github.com/settings/ssh 中

  • 点击New SSH key，title随意，key填入刚才复制的公钥，Add SSH key即可。

  • 回到Git Bash中检测GitHub公钥设置是否成功

    ssh git@github.com

    可能会出现以下错误：

    因为.ssh文件夹中缺少一个known_hosts文件，选择yes继续连接就会自动生成该文件。

    ssh会把每个访问过计算机的公钥(public key)都记录在known_hosts。当下次访问相同计算机时，OpenSSH会核对公钥。如果公钥不同，OpenSSH会发出警告， 避免受到DNS Hijack之类的攻击。

    文件补齐后，下次再登录时就可以进行正常操作啦~

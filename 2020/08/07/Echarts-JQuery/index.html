Echarts+JQuery

ECharts开源来自百度商业前端数据可视化团队，基于html5 Canvas，是一个纯Javascript图表库，提供直观，生动，可交互，可个性化定制的数据可视化图表。以官网的一个条形图 为例，记录一下echarts的摸索过程。

jsp文件中引入echarts.js

<script src="https://cdn.staticfile.org/echarts/4.3.0/echarts.min.js"></script>

添加按钮链接后续被调用的函数

<button type="button" class="btn btn-primary btn-sm" onclick="getBarEchart();">
<i class="ace-icon fa fa-plus bigger-110"></i>柱状图</button>

为图表准备一个Dom用于存储柱状图

<div id="container-bar" style="width: 600px;height:400px;"></div>

编写生成柱状图的函数。 其中关于option的配置，从官网查找想选择的图表将其样式复制到项目中即可。data可写静态数据，但大多数时候需要从需要从后台获取动态更新，这里使用的是JQuery的ajax()方法异步加载数据。

function getBarEchart() {
    // 获得对象的引用
    var dom = document.getElementById("container-bar");
    // 初始化echarts实例
    var myChart = echarts.init(dom);
    // 图表配置
    var option = {
        xAxis : {
            type : 'category',
            // data: ['Mon', 'Tue', 'Wed', 'Thu', 'Fri', 'Sat', 'Sun']
            data : jsonAge()
        },
        yAxis : {
            type : 'value',
            minInterval : 1
        },
        series : [ {
            // data: [120, 200, 150, 80, 70, 110, 130],
            data : jsonCount(),
            type : 'bar',
            showBackground : true,
            backgroundStyle : {
                color : 'rgba(220, 220, 220, 0.8)'
            }
        } ]
    };
    
    if (option && typeof option === "object") {
        // 使用刚指定的配置项和数据渲染生成
        myChart.setOption(option, true);
    }
}

页面：ajax请求(Asynchronous JavaScript and XML)

异步操作：通常指那些发生在幕后的，不会中断进程的操作。

function jsonAge() {
    var result = '';
    var url = basePath + "fra-test/Test.jsonAge.html";
    var params = null;
    $.ajax({
        "url" : url,
        "data" : params,
        "type" : "post",
        "async" : false,
        "datatype" : "json",
        "success" : function(json) {
            var obj = eval("(" + json + ")");
            result = obj;
        }
    });
    return result;
}

参数一览：

参数 描述
url 规定发送请求的 URL，默认是当前页面
data 规定要发送到服务器的数据
type 规定请求的类型（GET 或 POST）
async 布尔值，表示请求是否异步处理。默认是 true
dataType 预期的服务器响应的数据类型
success(result,status,xhr) 当请求成功时运行的函数

[补充] 跨站脚本攻击（XSS）

使用JSON时常见的安全漏洞，通常发生在js从服务器获取到一段JSON字符串并将其转化为js对象时。如果想要对代表对象的文本进行操作，首先要将其转换为对象并且装入内存中。

例：使用eval() 函数来将对象放入内存

var jsonString='{"animals":"cat"}';
var myObject=eval("("+jsonString+")");
alert(myObject.animals);

但是eval（）函数会将传入的字符无差别地编译执行，如果从第三方服务器中获取的JSON数据被替换成为恶意代码，那么我们的站点就会无辜蒙冤在访问者的浏览器中编译执行的代码。
JSON.parse() 便应运而生，该函数仅会解析JSON，不会执行脚本，更加安全可靠。

例：使用JSON.parse()代替eval（)

var jsonString='{"animals":"cat"}';
var myObject=JSON.parse(jsonString);
alert(myObject.animals);
